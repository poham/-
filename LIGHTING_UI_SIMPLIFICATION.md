# 燈光系統 UI 簡化改善

## 改善日期
2026-02-06 (第二版)

## 使用者反饋問題

### 1. 圖示與顏色問題
- ❌ 「棚」(藍色底)、「混」(紫色底)、「✓精準」(綠色框底) 的標籤顏色雜亂
- ❌ 無法達成視覺統一
- ❌ 介面過於複雜

### 2. 介面空間配置問題
- ❌ 右側已有 Protocol Deck，主欄的 Prompt 輸出顯得多餘
- ❌ 英文說明（如 "Rembrandt"）佔用空間
- ❌ 文字太多導致燈光圖示被縮得很小

### 3. 文字與互動問題
- ❌ 字體太小，閱讀困難
- ❌ 說明文字（如「臉頰處形成三角形光斑」）佔用空間
- ❌ Icon 被文字擠壓，看不清燈光圖案

## 解決方案

### 1. 視覺統一與簡化

#### 移除雜亂的顏色標籤
**改善前：**
```tsx
// 每個卡片上有多個顏色標籤
<div className="bg-blue-500/90">棚</div>
<div className="bg-green-500/90">✓ 精準</div>
<div className="bg-orange-500/90">~ 已調整</div>
```

**改善後：**
```tsx
// 只用小圓點表示狀態
{isActive && status === 'perfect' && (
  <div className="w-2 h-2 bg-green-400 rounded-full" />
)}
{isActive && status === 'modified' && (
  <div className="w-2 h-2 bg-orange-400 rounded-full" />
)}
```

#### 使用區塊分隔取代標籤
**改善前：**
```tsx
// 所有預設混在一起，用標籤區分
<div className="grid">
  {allSetups.map(setup => (
    <Card>
      <Badge>{setup.scenario === 'studio' ? '棚' : '混'}</Badge>
    </Card>
  ))}
</div>
```

**改善後：**
```tsx
// 分成兩個清晰的區塊
<div>攝影棚佈光</div>
<div className="grid">{studioSetups.map(...)}</div>

<div>混合模式（可用於戶外補光）</div>
<div className="grid">{hybridSetups.map(...)}</div>
```

### 2. 放大燈光圖示

#### 移除冗餘文字
**改善前：**
```tsx
<div className="p-5">
  <div className="aspect-square mb-3">
    <LightingPresetIcon />
  </div>
  <p className="text-[16px]">林布蘭光 (Rembrandt)</p>
  <p className="text-[11px]">臉頰處形成三角形光斑，經典戲劇感</p>
  <p className="text-[10px]">攝影棚 / 室內人像</p>
</div>
```

**改善後：**
```tsx
<div>
  <div className="aspect-square"> {/* 移除 mb-3，讓圖示更大 */}
    <LightingPresetIcon />
  </div>
  <div className="px-2 py-2"> {/* 減少 padding */}
    <p className="text-[14px]">林布蘭光</p> {/* 只保留中文名稱 */}
  </div>
</div>
```

#### 使用 Hover 顯示詳細說明
```tsx
<button
  title={setup.desc} // Hover 時顯示完整說明
  className="..."
>
  {/* 卡片內容 */}
</button>
```

### 3. 簡化資訊顯示

#### 移除 Prompt 輸出預覽
**改善前：**
```tsx
<div className="bg-slate-950/80 p-4">
  <span>Prompt 輸出</span>
  <div className="text-[11px] font-mono">
    {result.physicalDescription || result.geometryTags.join(', ')}
  </div>
  <div className="text-[11px] font-mono">
    風格: {result.styleTags.join(', ')}
  </div>
</div>
```

**改善後：**
```tsx
// 完全移除，因為右側已有 Protocol Deck
```

#### 簡化方向指示器
**改善前：**
```tsx
<div className="space-y-4">
  <div>模式狀態</div>
  <div>主光方向（3 行詳細資訊）</div>
  <div>陰影方向（2 行詳細資訊）</div>
  <div>Prompt 輸出預覽</div>
  <div>使用提示</div>
</div>
```

**改善後：**
```tsx
<div className="space-y-4">
  <div>模式狀態（簡化）</div>
  <div className="grid grid-cols-2">
    <div>主光：右前方 · 高角度</div>
    <div>陰影：落在左後方</div>
  </div>
  {/* 僅在風格繼承時顯示提示 */}
</div>
```

### 4. 網格佈局優化

**改善前：**
```tsx
// 5 欄佈局，卡片較小
<div className="grid grid-cols-2 md:grid-cols-3 xl:grid-cols-5 gap-2">
```

**改善後：**
```tsx
// 8 欄佈局，卡片更小但圖示更大（因為移除了文字）
<div className="grid grid-cols-4 md:grid-cols-4 xl:grid-cols-8 gap-2">
```

**邏輯：**
- 雖然欄數增加（5→8），但移除了大量文字和 padding
- 實際上圖示佔比從 ~40% 提升到 ~75%
- 整體視覺更清爽，圖示更清晰

## 改善成果

### 視覺統一
- ✅ 移除雜亂的顏色標籤
- ✅ 使用區塊分隔取代標籤
- ✅ 只用小圓點表示狀態（綠色=精準，橙色=已調整）

### 圖示放大
- ✅ 移除英文名稱（Rembrandt → 林布蘭光）
- ✅ 移除說明文字（使用 hover 顯示）
- ✅ 減少 padding（p-5 → px-2 py-2）
- ✅ 圖示佔比從 ~40% 提升到 ~75%

### 資訊簡化
- ✅ 移除 Prompt 輸出預覽（右側已有 Protocol Deck）
- ✅ 簡化方向指示器（2 欄並排顯示）
- ✅ 只在必要時顯示提示訊息

### 互動改善
- ✅ Hover 顯示完整說明（桌面版）
- ✅ 點選後在下方顯示詳細資訊
- ✅ 狀態指示清晰（小圓點）

## 技術細節

### 修改檔案
1. `components/lighting/LightingPresetGrid.tsx`
   - 移除篩選按鈕
   - 分成兩個區塊（攝影棚 + 混合模式）
   - 簡化卡片內容
   - 使用 `title` 屬性顯示 hover 說明
   - 調整網格佈局（5 欄 → 8 欄）

2. `components/lighting/LightingDirectionIndicator.tsx`
   - 移除 Prompt 輸出預覽
   - 簡化主光和陰影資訊
   - 使用 2 欄並排佈局
   - 減少文字描述

### 保留功能
- ✅ 狀態判斷邏輯（Perfect Match / Style Inheritance）
- ✅ 方向轉換邏輯（方位角 → 方向描述）
- ✅ 陰影計算邏輯
- ✅ 燈光圖示渲染

### 移除功能
- ❌ 情境篩選按鈕（改用區塊分隔）
- ❌ 卡片上的情境標籤（改用區塊標題）
- ❌ 英文名稱
- ❌ 說明文字（改用 hover）

## 結論

這次簡化改善大幅提升了燈光系統的可用性：
- **視覺更統一**：移除雜亂的顏色標籤
- **圖示更清晰**：放大燈光圖案，易於識別
- **資訊更精簡**：移除冗餘文字和重複資訊
- **互動更直觀**：使用 hover 和點選顯示詳細資訊

使用者現在可以快速瀏覽和選擇燈光預設，而不會被過多的資訊干擾。
