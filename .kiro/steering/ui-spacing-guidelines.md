# UI Spacing Guidelines

本文件定義 Nano Banana 專案中所有 UI 元素的間距規範，確保整體介面的一致性和緊湊性。

## 核心原則

1. **緊湊優先**：所有容器的 padding 應該貼合內容，避免過多的空白空間
2. **可讀性平衡**：在緊湊的同時，確保文字和元素有足夠的呼吸空間
3. **一致性**：相同類型的元素使用相同的間距規範

## Padding 規範

### 按鈕 (Buttons)

| 元素類型 | Padding | 使用場景 |
|---------|---------|---------|
| 主要按鈕 | `py-4` | 大型選項按鈕（如 Aspect Ratio） |
| 次要按鈕 | `py-2.5` | 中型功能按鈕（如預設角度） |
| 小型按鈕 | `py-1` | 小型控制按鈕（如 ZERO、RESET） |

### 標籤與徽章 (Labels & Badges)

| 元素類型 | Padding | 使用場景 |
|---------|---------|---------|
| 狀態標籤 | `px-4 py-1.5` | 顯示當前狀態（如 Physics Core、角度描述） |
| 數值顯示 | `px-6 py-2` | 顯示數值資訊（如光圈值 f/2.8） |

### 容器與卡片 (Containers & Cards)

| 元素類型 | Padding | 使用場景 |
|---------|---------|---------|
| 大型區塊 | `p-6` | 主要功能區塊（如 3D Gizmo 控制器） |
| 中型區塊 | `p-5` | 次要功能區塊（如自定義標籤） |
| 小型區塊 | `p-3` | 提示訊息框（如視覺提示） |
| 控制列 | `px-6 py-3` | 滑桿控制列（如垂直校準、滾轉） |

### 下拉選單與輸入框 (Selects & Inputs)

| 元素類型 | Padding | 使用場景 |
|---------|---------|---------|
| 大型選單 | `p-6` | 主要選擇器（如取景尺度） |
| 中型選單 | `p-4` | 次要選擇器 |

## 字體大小規範

### 提示文字 (Hint Text)

| 文字類型 | 字體大小 | 顏色 | 使用場景 |
|---------|---------|------|---------|
| 主要提示 | `text-[13px]` | `text-slate-300` | 重要說明文字 |
| 次要提示 | `text-[11px]` | `text-slate-400` | 輔助說明文字 |
| 標籤文字 | `text-[11px]` | `text-slate-400` | 欄位標籤 |

### 標題與內容

| 文字類型 | 字體大小 | 使用場景 |
|---------|---------|---------|
| 區塊標題 | `text-[14px]` | 主要功能區塊標題 |
| 子標題 | `text-[12px]` | 次要功能標題 |
| 按鈕文字 | `text-[14px]` | 主要按鈕 |
| 小按鈕文字 | `text-[10px]` | 次要/控制按鈕 |

## 間距調整規則

### 從舊規範遷移

如果發現元素的 padding 過大，按照以下規則調整：

1. **`py-8` → `py-4`**：大型按鈕和選項
2. **`p-8` → `p-6`**：大型容器區塊
3. **`py-4` → `py-3`**：控制列和中型元素
4. **`p-6` → `p-5`**：中型容器區塊
5. **`p-4` → `p-3`**：小型容器和提示框
6. **`py-3` → `py-2.5`**：中型按鈕
7. **`py-2` → `py-1.5`**：標籤和徽章

### 文字大小調整

如果發現文字過小或對比度不足：

1. **`text-[9px]` → `text-[11px]`**：最小可讀文字
2. **`text-[10px]` → `text-[11px]`**：提示文字
3. **`text-slate-600` → `text-slate-500`**：深色文字變亮一級
4. **`text-slate-500` → `text-slate-400`**：中等文字變亮一級
5. **`text-slate-400` → `text-slate-300`**：淺色文字變亮一級

## 特殊情況

### 文字溢出處理

當文字內容可能過長時：

1. 移除 `whitespace-nowrap`
2. 添加 `max-w-[XX%]` 限制最大寬度
3. 添加 `text-center leading-relaxed` 讓多行文字置中且行距舒適
4. 調整 `tracking` 值（如 `0.4em` → `0.3em`）減少字距

範例：
```tsx
// 修改前
<p className="text-[11px] font-black text-white uppercase tracking-[0.4em] whitespace-nowrap">
  {shotType}
</p>

// 修改後
<p className="text-[12px] font-black text-white uppercase tracking-[0.3em] text-center leading-relaxed">
  {shotType}
</p>
```

### 避免元素重疊

當視覺元素可能重疊時：

1. 調整 `bottom-X` 或 `top-X` 的數值
2. 使用 `z-index` 控制層級
3. 添加 `max-w-[XX%]` 限制寬度

範例：
```tsx
// 將底部文字框往下移，避免與內容重疊
<div className="absolute bottom-4 left-1/2 -translate-x-1/2 max-w-[90%]">
```

## 實施檢查清單

在調整任何 Section 的 UI 時，請檢查：

- [ ] 所有按鈕的 padding 是否符合規範
- [ ] 所有容器的 padding 是否符合規範
- [ ] 所有提示文字的大小和顏色是否符合規範
- [ ] 長文字是否有適當的溢出處理
- [ ] 元素之間是否有適當的間距，沒有重疊
- [ ] 整體視覺是否緊湊但不擁擠
