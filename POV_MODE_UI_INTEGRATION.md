# POV 模式 UI 整合總結

## 變更內容

將「特殊拍攝視角 (POV Mode)」整合到「快速預設角度」區塊中，避免介面層次過於複雜。

## 整合前後對比

### 整合前（獨立區塊）

```
┌─────────────────────────────────────────┐
│ 取景模式                                │
│ [自動] [商品] [人像]                    │
└─────────────────────────────────────────┘

┌─────────────────────────────────────────┐
│ 特殊拍攝視角 (POV)                      │  ← 獨立區塊
│ [下拉選單]                              │
└─────────────────────────────────────────┘

┌─────────────────────────────────────────┐
│ 即時觀景模擬                            │
└─────────────────────────────────────────┘

┌─────────────────────────────────────────┐
│ 3D 攝影機控制器                         │
│ ├─ 快速預設角度                         │
│ ├─ 攝影模式                             │

```

### 整合後（統一容器）

```
│ 取景模式                                │

│ ├─ 快速預設角度                         │
│ │   [正面] [左側] [右側] [背面]         │
│ ├─────────────────────────────────────  │  ← 分隔線
│ ├─ 攝影模式                             │

```

## 優點

1. **視覺清晰**：減少獨立區塊數量，介面更緊湊
2. **邏輯一致**：POV、預設角度、攝影模式都是「調整攝影機」的功能，放在一起更合理
3. **易於理解**：用戶不會混淆「取景模式」和「特殊視角」的差異
4. **操作順暢**：從上到下依序為「特殊視角 → 預設角度 → 攝影模式 → 數值微調」

## 技術細節

- 使用 `border-t border-slate-700/50` 分隔線區分功能區塊
- POV 模式放在最上方，因為它在提示詞中的優先級最高
- 保持 `p-5` 的容器 padding，符合 UI spacing guidelines
- 選擇 POV 模式後立即顯示提示詞預覽
